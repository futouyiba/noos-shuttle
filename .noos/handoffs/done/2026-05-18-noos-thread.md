# 交接：钓鱼玩法场次与经济框算表设计
## 意图
将一次关于“玩法场次、鱼池结构、RTP、跑刀/猛攻策略、搏金经济”的讨论，整理成可执行的表格设计任务，交给 Codex / Claude Code / OpenCode 等代理继续产出 Markdown、Excel 或结构化表格。
目标是帮助用户明天把 CEO 和执行制作人较抽象的想法转化为可见、可讨论、可推演的表格，避免继续停留在口头判断和鸡同鸭讲。
## 背景摘要
当前会议结论确认：后续不再只按鱼重量或等阶来讨论场次，而是按照“玩法”区分虚拟场次。每个玩法内可以包含多个重量段，不同玩法之间的重量段也可以重合。
经济推演上，可以先参考《俄罗斯钓鱼4》的鱼价体系，再结合本项目时间节奏和货币转换关系，推算货币量级、单位时间收益、资产积累规模。
场次定位上，可以参考《三角洲行动》的风险收益结构：玩家可以在低风险场次赚钱，再去更高风险场次尝试；亏损后可以回到低风险场次重新积累。类比到钓鱼中，“跑刀”可对应便宜饵、低成本、低风险、主要钓杂鱼/保底鱼；“猛攻”可对应贵饵、高成本、提高目标鱼/稀有鱼权重，追求更高收益上限。
搏金系统不必保证所有玩家都能找到一个稳定微亏的等阶。可以设置标准 RTP，例如 98%，但允许高手因操作、策略、付费或风险选择不同，长期达到 110%～120% 甚至更高的实际 RTP，并一路推进到高阶玩法。
## 任务
请继续执行以下任务：
- 设计一组“玩法场次与经济框算表”的 Markdown 表格或 Excel sheet 结构。
- 表格不需要一次性覆盖全游戏，优先做一个最小可行切片。
- 建议至少包含以下 5 张表： - 玩法场次总表
- 鱼池结构表
- 饵 / 策略修正表
- RTP 基础框算表
- 玩家水平修正表
- 每张表需要包含字段、字段说明、示例行。
- 表格要能支持 CEO / 执行制作人 / 数值侧围绕同一组数据继续讨论。
- 如输出 Excel，可以每张表对应一个 sheet；如输出 Markdown，需保持表格清晰、字段命名统一。
## 约束
- 不要一开始追求最终数值准确，第一版目标是建立讨论结构。
- 不要把所有内容塞进一张大表，应拆成多张职责清晰的表。
- 第一版可以假设所有鱼的单价一致，用重量差异代表收益期望差异。
- 真实鱼种先作为“语义锚点”，不要直接作为最终计算核心。
- 表中应同时保留： - 暂定鱼种锚点：如鲫鱼、鲤鱼、鲈鱼、鳜鱼、鳟鱼、石斑、金枪鱼、旗鱼等
- 正式鱼种占位：如鱼A、鱼B、鱼C，用于后续替换和计算
- 姥鲨一类大型海洋生物可作为奇观感/体型锚点，但不建议作为普通可钓鱼池的常规目标。
- 标准 RTP 是系统基准，不代表所有玩家实际结果。
- 需要明确区分： - 基础收益倍率
- 极限爆发倍率
- 波动性
- 标准 RTP
- 实际玩家 RTP 区间
- 跑刀、稳健、猛攻应主要通过饵成本、鱼池权重修正、咬钩率修正、失败损失等参数表达。
- 高手正收益不必视为问题，应在表中作为一种允许的玩家分层结果表达出来。
## 验收标准
输出结果应满足：
- 用户可以直接拿去开会或继续加工成 Excel。
- 每张表的用途明确，能回答一个核心问题。
- 字段足够具体，后续能填数，不只是概念说明。
- 示例行能体现以下设计逻辑： - 同一玩法场次内有杂鱼、保底鱼、目标鱼、稀有鱼、巨物/大奖
- 不同饵/策略会改变鱼池权重和成本
- 标准玩家可按 98% RTP 框算
- 新手可能落到 70%～85% RTP
- 普通玩家约 95%～100% RTP
- 熟练玩家可能 105%～115% RTP
- 高手玩家可能 115%～125% RTP
- 表格应能支持讨论以下问题： - 这个场次是稳定赚钱还是高波动搏金？
- 目标鱼和稀有鱼分别承担什么经济作用？
- 跑刀和猛攻的差异到底来自成本、鱼池权重还是失败损失？
- 经济失控时应调鱼价、鱼权重、饵成本、维修成本还是门票？
- 高手一路正收益冲高阶是否被允许？
- 行文应简洁、清楚、偏会议材料风格，不要写成长篇散文。
## 建议给下一位代理的指令
请基于以下结构继续产出一份 Markdown 文档或 Excel 方案：
- 先写一个简短总览，说明这套表的目标不是算准最终数值，而是将玩法场次、鱼池、策略、RTP 和玩家水平差异显性化。
- 然后依次输出 5 张表： 第一张：玩法场次总表
目的：定义每个玩法场次的体验定位、重量段、基础收益倍率、波动性、极限倍率、策略类型。 第二张：鱼池结构表
目的：定义一个场次内的杂鱼、保底鱼、目标鱼、稀有鱼、巨物/大奖，以及它们的权重、重量、价值和经济作用。 第三张：饵 / 策略修正表
目的：表达跑刀、稳健、猛攻三种策略如何改变成本、鱼池权重、咬钩率和风险。 第四张：RTP 基础框算表
目的：用鱼获期望价值、饵成本、门票/船费、体力成本、维修成本、失败损失期望，框算标准 RTP 和单位时间净收益。 第五张：玩家水平修正表
目的：表达新手、普通玩家、熟练玩家、高手玩家在跑鱼率、断线/断竿率、维修损耗倍率、鱼获效率倍率和实际 RTP 区间上的差异。
- 每张表后面补充 3～5 条字段解释。
- 最后补充“明天会议建议起手话术”，用于用户向 CEO 和执行制作人说明这组表的作用。
可以使用以下简化公式：
单条价值 = 平均重量 × 单价系数鱼获期望价值 = Σ（鱼i权重 × 鱼i单条价值）总投入 = 饵成本 + 门票/船费 + 体力成本 + 维修成本 + 失败损失期望标准RTP = 鱼获期望价值 / 总投入
第一版可以使用示例数值，但必须标注为“示例值 / 待校准”。
## 未决问题
- 第一版切片应选哪个玩法：湖泊浮钓、港口近海、溪流路亚，还是大海图玩法？
- 标准 RTP 是否统一设为 98%，还是不同玩法场次允许不同基准？
- 基础收益倍率和极限倍率的具体范围需要 CEO / 执行制作人确认。
- 高手玩家长期 115%～125% RTP 是否可接受？可接受到哪个阶段？
- 跑刀、稳健、猛攻是否只通过饵表达，还是还要叠加钓具、鱼卡、技能、门票等策略入口？
- 真实鱼种库是否需要单独做深度调研，整理“钓鱼界知名鱼种 / 玩家认知强 / 适合玩法 / 体型区间 / 可承担经济角色”的表？
- 鱼卡系统后续如何接入 RTP、鱼池权重和经济调节？
- 是否需要进一步做 Monte Carlo / 计算机模拟，预演不同玩家水平下的长期收益分布？
